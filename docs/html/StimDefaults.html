Parameter Defaults¶

class StimProgram.StimDefaults(shape='circle', fill_mode='uniform', orientation=0, size=None, outer_diameter=75, inner_diameter=40, check_size=None, num_check=64, delay=0, duration=0.5, location=None, timing='step', intensity=1, alpha=1, color=None, color_mode='intensity', image_channel='all', fill_seed=1, move_seed=1, speed=10, num_dirs=4, start_dir=0, start_radius=300, travel_distance=50, ori_with_dir=False, intensity_dir='both', sf=1, phase=None, phase_speed=None, contrast_channel='green', movie_filename=None, movie_size=None, period_mod=1, image_size=None, image_filename=None, table_filename=None, trigger=False, move_delay=0, num_jumps=5, jump_delay=100, force_stop=0)[source]¶

Bases: object

Super class to hold parameter defaults. GUI passes dictionary of all parameters, whether used to make stim or not.

Parameters:
  • shape (string) – Shape of the stim, ‘circle’, ‘rectangle, or ‘annulus’.
  • fill_mode (string) – How the stim is filled. Can be ‘uniform’, ‘sine’, ‘square’, ‘concentric’, ‘checkerboard’, ‘random’, ‘image’, or ‘movie’.
  • orientation (float) – Orientation of the stim, in degrees.
  • size (list) – Size of the stim, as an x, y list.
  • outer_diameter (float) – Size of circle, or outer diameter of annulus, in micrometers.
  • inner_diameter (float) – Inner diameter of annulus, in micrometers.
  • check_size (list) – Size of each check in a checkerboard or randomly filled board, as an x, y list in micrometers.
  • num_check (int) – The number of checks in each direction.
  • delay (float) – The time to between the first frame and the stim appearing on screen. Rounds up to the nearest frame.
  • duration (float) – The duration for which the stim will animated. Rounds up to the nearest frame.
  • location (list) – Coordinates of the stim, as an x, y list.
  • timing (string) – How the stim appears on screen over time. For ‘step’, the stim appears immediately. Other options include ‘sine’, ‘sawtooth’, ‘square’, and ‘linear’.
  • period_mod (float) – For cyclic timing modes, the number of cycles.
  • alpha (float) – The transparency of the of the stim, between 0 and 1. Does not apply to checkerboard or random fill stims, or images and movies.
  • color_mode (string) – The way in which the color of the stim is determined. Can be either ‘intensity’, or ‘rgb’.
  • contrast_channel (string) – The color channel in which color is displayed in intensity mode. For the RGB color mode, contrast channel is only used if a fill other than uniform is specified.
  • intensity (float) – The color of the stim relative to background, between -1 and 1. For fills, the color will fluctuate between high and low values of the specified intensity relative to background, and thus background values on either extreme must use low intensities or risk hitting the color ceiling/floor of the monitor.
  • color (list) – RGB color values of the stim, as a list of 3 values between -1 and 1.
  • fill_seed (float) – The seed for the random number generator for random fills.
  • move_seed (float) – The seed for the random number generator for random movement.
  • speed (float) – The speed of moving stims, in micrometers per second.
  • num_dirs (int) – The number of directions radially moving stims will travel in.
  • start_dir (float) – The start direction for radially moving stims.
  • start_radius (float) – The start radius for radially moving stims.
  • move_delay (float) – The amount of wait time between each move direction.
  • travel_distance (float) – The distance that randomly moving stims will travel before choosing a new direction.
  • sf (float) – The spatial frequency of a stim texture.
  • phase (list) – The offset of texture in the stim. Units are in cycles, so integer phases will result in no discernible change.
  • movie_filename (string) – File path of the movie to be displayed.
  • movie_size (list) – Size of the movie, as an x, y list, in micrometers. Keep aspect ratio or movie will be distorted.
  • image_filename (string) – File path of the image to be displayed.
  • image_size (list) – Size of the image, as an x, y list, in micrometers. Keep aspect ratio or movie will be distorted.
  • table_filename (string) – File path of the table to be used for coordinates.
  • trigger (bool) – Whether or not to send a trigger for the stim.
  • num_jumps (int) –
  • jump_delay (float) –
  • force_stop (float) – time at which stim should end, overrides all other timing. Useful for moving and table stims.
